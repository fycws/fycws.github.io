Why is OPSR-JA Almost double the size of any other resource pack?

OPSR-JA Follows a quality textured and photorealism way which already increases the overall pack size a lot but OPSR-JA also provides a built-in backup for linux based systems which doubles the pack size.

Drivers of OPSR-JA

OPSR-JA has built-in drivers which are responsible for providing every service to you for example the service could be updating, backing up or anything else. The drivers are extremely important for this resource and the drivers are also fully open source and safe to use and even modify.

Check the "/src/drivers/code/" for drivers source code

Can we update drivers?

Yes, OPSR-JA Provides drivers separately in releases meaning you may be able to update your OPSR-JA drivers with ease.

How to update drivers?

If you want to update OPSR-JA drivers then you must be using a linux based system because OPSR-JA Only provides drivers for linux based systems only. You must be familiar with CLI tools of linux and terminal of your operating system.

Suggested Systems:

  • Debian
  • Ubuntu
  • Arch
  • Manjaro

Don't worry even if your distribution is different but if it is linux based then it may run drivers as well.

Let's suppose you are using a linux based system and want to update the drivers of OPSR-JA then follow the steps but do not follow them if you are using Windows or MAC or anything else.

  • Locate the Pack file (Typically in "/home/[user]/.minecraft/resourcepacks/")
  • Extract the Resource pack file to a different location (e.g.: "/home/Documents/RP/")
  • Extract the pack.
  • Locate your drivers.
  • Extract your Drivers.
  • Copy your drivers all files and folders.
  • Paste the copied content in the root directory of the extracted resource pack and if it asks for replacing then replace the old drivers with new.
  • Re-brand and Compress the content after copying done.
  • Paste your resource pack in your resourcepacks directory.

Congratulations!! You have successfully updated your old drivers.

What if I do not update drivers?

OPSR-JA never forces to update the drivers always because drivers are not designed for running the resource pack but for providing technical features for resource pack for example, upscaling, backup etc. If you won't update your drivers then it's totally fine your pack will still run fine as expected and updating drivers also means the new texture files must be added otherwise it may result bugs in-game. So generally updating drivers is good for accessing new features only but updating drivers in an older version of resource pack will definitely leave bugs in game. So do not update them until you really want to access new features because the drivers released with every updated versions of the resource pack and your OPSR-JA will already have its stable drivers and those drivers are designed to work with your version of OPSR-JA means they will run extremely fine and smoothly where new drivers in old versions may lead to glitches or bugs or even crash your game.

In which language are the drivers written?

OPSR-JA uses a combination of programming languages but all of them are low level because most of the part of drivers of OPSR-JA are written in C++ but some of them are written in Zile which is a custom programming language of ghgltggamer designed to be low level. C++ and Zile are due to performance and Zile itself transpiles into C++ during its own compilations means speed of drivers are insanely good.

As mentioned previously all the drivers are safe and open source means everyone can inspect and modify the driver's core with ease.

Minecraft has a huge audience in windows but drivers of OPSR-JA do not work in windows because they are linux compiles and designed to work with linux systems effectively. MACOS restricts the applications a lot which is not good for our drivers as our drivers need to be called manually by the user. Again safety issues are the main reason because we want to provide the safest experience to our families which MS and Apple do not want. Drivers and the whole resource pack is completely open source you can review any part any time you want.

How to backup OPSR-JA?

If you want to backup OPSR-JA then you must be using a linux based system because OPSR-JA Only provides backups for linux based systems only. You must be familiar with CLI tools of linux and terminal of your operating system.

Suggested Systems:

  • Debian
  • Ubuntu
  • Arch
  • Manjaro

Don't worry even if your distribution is different but if it is linux based then it may run backup program as well.

Let's suppose you are using a linux based system and want to backup OPSR-JA then follow the steps but do not follow them if you are using Windows or MAC or anything else.

  • Locate the Pack file (Typically in "/home/[user]/.minecraft/resourcepacks/")
  • Extract the Resource pack file to a different location (e.g.: "/home/Documents/RP/")
  • Extract the pack.
  • Open the terminal in the extracted folder
  • Fire up the backup drivers by the following command
    • ./openlogs driver-backup
  • And wait until you see "backup completed with status[1]"
  • In case you find any permission error then use the following commands.
    • sudo chmod +x openlogs
./pack_backup.opsrja
  • If you encounter a path error then check out your terminal path.
  • If you encounter any other type of error then it may be because of broken backup which might be due to damage to OPSR-JA backup system by minecraft or by you. Check whether your OS supports Executable System of backup or not.
  • Once the backup is done rebrand and compress the resource pack and enjoy it again!

Why is the backup system only available for Linux?

Our backup system is only for linux because Windows CPP Compiles were not possible as G++ was not officially available for windows and there is a 3rd party tool for it called MinGW which allows to use g++ on windows but it's first of all 3rd party and secondly it's too minimalist in terms of design and portability where official G++ is of course really a robust compiler which can compile any part of our backup system as our backup system is primarily written in C++ with Zile based drivers. MAC-OS is restricted a lot which is why it's a thing of trouble for devs to bring backup system while ensuring compatibilities with other users.
